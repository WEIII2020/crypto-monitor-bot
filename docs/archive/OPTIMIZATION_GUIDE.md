## 问题 1️⃣：应该监控哪些币种？

### 🎲 庄家最喜欢操控的币种特征

| 特征 | 理想范围 | 原因 |
|-----|---------|------|
| **市值** | $50M - $500M | 太小流动性差，太大拉不动 |
| **24h 交易量** | $5M - $50M | 有人气但不过热 |
| **价格** | $0.01 - $10 | 散户心理喜欢"便宜"币 |
| **上币时间** | 3个月 - 2年 | 新币波动大，老币关注少 |
| **交易所** | Binance 现货 | 流动性最好，散户最多 |

### 🚫 应该避免的币种

- ❌ **超大盘**：BTC、ETH（你已经在监控，但不是主要目标）
- ❌ **超小盘**：市值 < $10M（流动性差，容易归零）
- ❌ **稳定币**：USDT、USDC（无操控空间）
- ❌ **交易量极低**：24h < $1M（没有散户接盘）

## 🚀 立即可以做的优化

### 优化 1：动态币种池（推荐）

**目标：** 每天自动筛选符合条件的币种

**实现步骤：**
1. 每天 00:00 调用 Binance REST API 获取所有交易对
2. 按市值和交易量筛选
3. 更新监控列表
4. 重新订阅 WebSocket

**代码位置：** 创建 `src/utils/symbol_selector.py`

**API 调用：**
```python
# Binance REST API（公开，无需 Key）
GET https://api.binance.com/api/v3/ticker/24hr

返回所有交易对的 24h 统计，可以直接用于筛选
```

**预期效果：**
- 监控 50-100 个币种
- 每周捕捉 5-10 次明显的庄家操作
- 告警准确率 > 60%

### 优化 2：告警优先级

**问题：** 监控太多币种，告警会很吵

**解决方案：**
```python
告警分级：
- 🔥 紧急：Tier 3/4 币种 + CRITICAL 信号
- ⚠️ 重要：Tier 2 币种 + CRITICAL 信号  
- ℹ️ 提示：Tier 1 币种或 WARNING 信号

Telegram 推送策略：
- 紧急：立即推送 + 声音
- 重要：立即推送（静音）
- 提示：每小时汇总一次
```

### 优化 3：历史回测

**目标：** 验证检测规则的准确性

**方法：**
1. 收集 1-2 周的历史数据
2. 回放检测算法
3. 统计准确率：
   - 真阳性：确实是庄家操作
   - 假阳性：误报
   - 假阴性：漏检

**调优参数：**
```python
# 可调整的阈值
volume_spike_threshold = 5.0      # 交易量倍数
sideways_threshold = 3.0          # 横盘定义
significant_rise = 10.0           # 显著上涨
```

根据回测结果调整这些参数。
